# Change: 增强每周智能体深度洞察报告的信息量与分析深度

## Why
当前生成的每周深度洞察报告中，论文和资讯部分提炼的信息量太少：
- **论文部分**：仅显示功能、标题和简短核心内容（限制130字符），缺少作者、机构、发布时间、详细摘要等关键信息
- **资讯部分**：缺少发布时间显示、产品分析不足、摘要内容被截断（限制180字符），未能充分利用每日日报中的完整摘要信息
- **综合洞察**：LLM 提示词过于简单，仅提供标题列表，缺少上下文信息（作者、机构、摘要、发布时间、产品等），导致生成的洞察不够深刻和详细

这些限制导致报告的价值和可读性下降，无法满足深度分析的需求。

## What Changes
- **论文部分增强**：
  - 在表格中添加发布时间、作者/机构信息列（如有）
  - 增强核心内容提取逻辑，使用 LLM 从更丰富的上下文（包括标题、标签、作者、机构等）提取更详细的描述
  - 优化 LLM 提示词，要求提取论文的关键贡献、创新点、应用场景等更深入的信息
  - 核心内容长度限制从130字符扩展到250字符，允许更详细的描述
  
- **资讯部分增强**：
  - 在表格中添加发布时间列，显示资讯的发布时间（published_at，格式化为 YYYY-MM-DD）
  - 增强产品分析功能，提取并展示每条资讯涉及的具体产品名称（而非仅统计数量）
  - 完整使用从 markdown 日报中提取的摘要信息，不再截断到180字符，而是完整展示（但限制最大500字符以避免过长）
  - 添加产品列，展示每条资讯涉及的主要产品（如 "OpenAI ChatGPT", "小米 MiMo" 等）
  - 优化摘要提取逻辑，优先从 markdown 文件中提取完整摘要，而不是从 tags 中提取

- **综合洞察增强**：
  - 大幅增强 LLM 提示词，提供更丰富的上下文信息：
    - 论文：标题、作者、机构、标签、发布时间、排名/评分
    - 资讯：标题、发布时间、来源、产品、完整摘要、标签
    - SDK：仓库、版本、发布时间、主要变更详情
  - 要求 LLM 生成更深入的洞察：
    - 趋势洞察：短期（本周）、中期（近月）、长期（趋势方向）三个层次
    - 主题聚类：识别并分析跨领域的主题模式
    - 影响分析：技术影响、市场影响、产业影响
    - 机会识别：基于数据和趋势识别的机会点
    - 风险评估：潜在风险和挑战
  - 优化提示词结构，采用结构化输出要求，确保生成的洞察具有逻辑性和可读性

## Impact
- Affected specs: `specs/weekly-intel-report/spec.md` (需要 MODIFIED 相关需求)
- Affected code:
  - `report/report_writer.py`: 修改论文和资讯表格结构，添加新列
  - `report/processors.py`: 增强 `extract_paper_details` 和 `identify_important_news` 函数，提取更多信息
  - `report/readers.py`: 优化摘要提取逻辑，确保完整获取 markdown 中的摘要
  - `report/insights.py`: 大幅增强 `build_llm_prompt` 和 `generate_insights_llm` 函数，提供更丰富的上下文和更深入的洞察要求
- 不破坏现有功能：所有增强都是向后兼容的，不会影响现有报告生成流程
