# Change: 为报告生成添加图片生成功能

## Why

当前报告生成器（`report`）仅生成纯文本的 Markdown 报告，包含 Mermaid 图表。为了提升报告的可视化效果和信息传达能力，需要添加图片生成功能，使报告能够包含自定义生成的 SVG 图片或从 PPT 导出的图表，形成图文结合的 Markdown 报告。

## What Changes

- **新增图片生成能力**：在报告生成过程中，使用大模型判断哪些内容适合生成图片（如趋势图表、架构图、数据可视化等）
- **信息提取**：从报告内容中提取关键信息，用于图片生成
- **集成现有工具**：
  - 优先使用 `svg_generator` 模块生成 SVG 图片并嵌入 Markdown
  - 可选使用 `PPT` 模块功能生成图表（如需要更复杂的图表类型）
- **图文结合报告**：生成的 Markdown 报告中包含图片引用，形成图文结合的展示效果

## Impact

- **受影响的规范**：`weekly-intel-report`（修改现有报告生成能力）
- **受影响的代码**：
  - `report/main.py`：添加图片生成逻辑
  - 可能复用 `svg_generator/src/llm_service.py` 和 `svg_generator/src/main.py`
  - 可选择性集成 `PPT` 模块功能
- **依赖关系**：需要 `LLM_API_KEY` 配置（用于判断和生成图片）
- **输出格式**：报告输出目录需要包含 `assets/` 子目录用于存储生成的图片文件
